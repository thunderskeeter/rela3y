## Messaging Boundary Classifications
- `POST /webhooks/sms`: production-critical, inbound, synchronous, external-side-effecting, tenant-safe after webhook tenant resolution, coupled to opportunities/actions via `handleSignal()`, deterministic key available via `MessageSid` / claimed webhook id.
- Missed-call processing (`/webhooks/missed-call`, `/webhooks/voice/dial-result`): production-critical, inbound, synchronous, external-side-effecting, tenant-safe after webhook tenant resolution, coupled to opportunities/actions via `handleSignal()`, deterministic key available via `CallSid` / claimed webhook id.
- `POST /webhooks/event` booking-created / lead-lost conversation sync: production-critical, inbound/internal, synchronous, external-side-effecting for booking confirmation, tenant-safe after webhook tenant resolution, coupled to opportunities/actions via `handleSignal()`, deterministic key available via explicit/derived webhook event id.
- `POST /api/conversations/:id/send` and `POST /api/send`: production-critical, outbound, synchronous, external-side-effecting, tenant-safe from authenticated route tenant context, coupled to opportunity sync through deferred `attemptOutboundMessage()` behavior, deterministic key available when caller carries `actionId` / `runId` + `stepId` / correlation metadata.
- `POST /api/conversations/:id/status`: production-critical, internal-only, synchronous, tenant-safe from authenticated route tenant context, messaging-only, no deterministic idempotency key today.
- `POST /api/bookings/manual`: production-critical, internal-only, synchronous, tenant-safe from authenticated route tenant context, coupled to opportunity handling through downstream booking/invoice flows, deterministic key available via generated booking id.
- `publicBooking.syncBookingToConversation()`: production-critical, inbound/internal, synchronous, external-side-effecting through downstream automation/invoice flows, tenant-safe from booking token account resolution, coupled to opportunities/actions via `handleSignal()`, deterministic key available via booking id.
- `flowEngine.startFlow()` / `advanceFlow()`: production-critical helper path, internal-only, synchronous, tenant-safe when tenant is passed correctly, messaging-only, no deterministic idempotency key today.
- `automationEngine.sendAutomationMessage()`: production-critical async/orchestrator edge, outbound, asynchronous, external-side-effecting, tenant-safe from rule context + DB SQL scope, coupled to opportunities/actions indirectly through automation follow-up state, deterministic key available when automation metadata carries rule/template correlation.
- `actionExecutor.executeRevenueAction()` send branch: production-critical async/orchestrator edge, outbound, synchronous orchestration, external-side-effecting, tenant-safe from opportunity/action tenant context + DB SQL scope, coupled to opportunities/actions directly, deterministic key available via `actionId`.
- `agentEngine.executeStep()` SEND_MESSAGE branch: production-critical async/orchestrator edge, outbound, asynchronous or scheduled, external-side-effecting, tenant-safe from run/opportunity tenant context + DB SQL scope, coupled to opportunities/actions directly, deterministic key available via `runId` + `stepId`.
- `POST /webhooks/twilio/status`: production-critical provider callback edge, inbound, synchronous, external-side-effecting, tenant-safe only when tenant `to` is carried on the callback URL or other signed selector, messaging-only, deterministic key available via `MessageSid` + `MessageStatus` + `ErrorCode`.
- `messageRetryService.runRetryBatch()`: production-critical lifecycle-hardening edge, background/internal, asynchronous, external-side-effecting, tenant-safe from claimed DB rows, messaging-only, deterministic identity via canonical `message.id` and current row lock semantics.
- `runComplianceRetentionPurge()`: deferred background/internal edge, tenant-scoped by `to`, not yet boundary-owned, no deterministic idempotency key.
- `dev.routes.js` conversation/message mutation helpers: dev-only, mixed sync helper paths, tenant-risk remains JS-only, explicitly deferred.
